# Velox UI Designer Skill

## Purpose

This skill helps Codex design clean, premium, strategic, and practical user interfaces for Velox Creator Intelligence.

Velox is an AI creator intelligence system that discovers emerging creators, analyzes their growth, simulates campaign outcomes, and recommends how to monetize and scale them.

The interface should help users understand creator opportunity quickly.

Velox should feel like an intelligence dashboard, not a social media feed.

---

## Core UI Principle

Clarity beats decoration.

Do not overdesign. Do not add visual effects that make the system harder to understand.

The Velox interface should be simple, useful, professional, and easy to navigate.

---

## Design Personality

Velox UI should feel:

- Premium
- Intelligent
- Clean
- Modern
- Strategic
- Professional
- Calm
- Structured
- Easy to understand

The interface should create confidence without feeling complicated.

## Preferred UI Patterns

Use:

- Sidebar navigation
- Clean cards
- Metric panels
- Search bars
- Filters
- Tables
- Score badges
- Section headers
- Insight summary blocks
- Report-style layouts
- Simple charts where useful
- Responsive grids
- Clear call-to-action buttons

Avoid:

- Too many colors
- Heavy gradients
- Random animations
- Overloaded pages
- Confusing navigation
- Decorative elements with no purpose
- Complex interactions in Version 1
- Social-feed-style layouts unless specifically required

## Dashboard Layout Guidance

The dashboard should communicate value quickly.

It should show:

1. Total creators tracked
2. Average Velox score
3. Fastest growing creator
4. Top-performing niche
5. Campaign simulations
6. Recent reports
7. Top creator opportunities
8. Watch items or risks
9. Intelligence summary
10. Recommended next action

Use metric cards, simple tables, and insight panels.

The dashboard should answer:

“What is Velox seeing right now?”

---

## Creator Table Guidance

Creator tables should be clean and useful.

Recommended columns:

- Creator name
- Platform
- Niche
- Country
- Followers
- Average views
- Engagement rate
- Growth rate
- Velox score
- Status
- View profile action

Add search and filters when requested.

Filters may include:

- Platform
- Niche
- Country
- Minimum Velox score
- Monetization status
- Brand readiness status

Keep the table readable on smaller screens.

---

## Creator Profile Layout Guidance

Creator profiles should feel like intelligence briefs.

Recommended sections:

1. Creator overview
2. Key metrics
3. Velox score
4. Score breakdown
5. Growth insight
6. Engagement insight
7. Brand fit notes
8. Monetization readiness
9. Creator opportunity
10. Risks and watch items
11. Recommended next action
12. Admin notes

The user should leave the page knowing why the creator matters and what should happen next.

---

## Score Badge Guidance

Score badges should make scoring easier to interpret.

Use these score labels:

- 85–100: High Potential
- 70–84: Strong Opportunity
- 55–69: Developing
- 40–54: Needs Support
- Below 40: Watch Carefully

Do not rely only on color. Include clear text labels.

---

## Campaign Simulation UI Guidance

The Campaign Simulation page should include a simple form and a clear result panel.

Form inputs may include:

- Brand name
- Campaign goal
- Target audience
- Budget range
- Preferred platform
- Required niche
- Selected creator
- Campaign duration

Result outputs may include:

- Estimated reach
- Estimated engagement
- Conversion potential
- Risk level
- Brand fit rating
- Recommendation
- Suggested next action

The page should feel like a decision tool, not just a form.

---

## Report Layout Guidance

Reports should feel client-ready and professional.

A report preview should include:

1. Report title
2. Executive summary
3. Creator overview
4. Platform and niche
5. Key metrics
6. Velox score
7. Score breakdown
8. Growth insight
9. Engagement insight
10. Brand fit analysis
11. Monetization readiness
12. Risks
13. Recommended next action
14. 30-day strategic recommendation

Make report previews readable when printed or exported as PDF.

Hide navigation elements in print mode if possible.

---

## Settings Page Guidance

The Settings page should not become complicated in Version 1.

Use placeholder sections for:

- Project status
- Data source settings
- Scoring settings
- AI settings
- Supabase settings
- Cloudflare settings

Clearly label features that are planned but not connected yet.

---

## Tailwind CSS Guidance

Use Tailwind CSS for layout and styling.

Prefer:

- Consistent spacing
- Responsive grids
- Rounded cards
- Subtle shadows
- Clear typography hierarchy
- Strong contrast
- Clean table styling
- Simple hover states

Avoid introducing a new UI library unless specifically requested.

Use simple class structures and readable components.

---

## Responsiveness Rules

Design for desktop first, but keep layouts usable on tablets and mobile.

Use:

- Responsive grids
- Scrollable tables when necessary
- Stacked cards on small screens
- Collapsible or simplified navigation when needed

Do not sacrifice clarity for visual complexity.

---

## UX Writing Rules

Use clear, simple labels.

Prefer:

- “Growth Score” instead of “Growth Velocity Index”
- “Recommended Next Action” instead of “Strategic Intervention Output”
- “Brand Fit” instead of “Commercial Compatibility Matrix”
- “Campaign Simulation” instead of “Predictive Campaign Intelligence Engine”
- “Creator Opportunity” instead of “Opportunity Intelligence Module”

Velox can be powerful without sounding complicated.

---

## Accessibility Guidance

Use readable text sizes.

Ensure good contrast.

Use labels for form inputs.

Use buttons and links clearly.

Do not communicate important meaning only through color.
